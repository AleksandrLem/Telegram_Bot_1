Git Hub
уже созданные локальные и удаленные хранилица связываем так:
git remote add произвольное-имя (обычно origin) url-хранилища

чтобы просмотреть список имён уже привязанных хранилищ,
git remote show

для колнирования удаленного хранилища
сначала заходим в папку, куда будем клонировать репозиториз из github
git clone <ссылка_на_репозиторий>

заходим в папку с уже клонированным репозиторием, вводим команду
git remote -v - покажет две ссылки на репозиторий (http or ssh)
git remote set-url origin <ссылка_на_репозиторий_для_доступа_по_ssh>

git push origin main - отправляем commit на git_hub

Команды для выполнения обновлений с git_hub
git branch - просмотреть текущую ветку
git branch new_branch - создать новую ветку
git checkout new_branch - перейти в новую ветку
git checkout -b main - создать новую ветку main и сразу перейти на нее
git fetch
git merge
Вместо двух команд выше можно использовать одну:
git pull
git pull <remote> <branch> - или так, где <remote> - <удаленный> репозиторий
git branch --set-upstream-to=<внешний-репозиторий>/<внешняя-ветка> <локальная-ветка>
git remote remove [имя репозитория] - отключение от веншнего репозитория

PostgreSQL
-----------------------
service postgresql status - проверка работы СУБД
service postgresql - покажет команды, которые можно выполнить

sudo -i -u postgres --- вход в postgresql
psql ------------------  вход в консоль БД

\l - список баз данных
q - выход
\q - выйти из консоли psql
createdb имяБД - создает БД
clear - очищает экран от предыдущих шагов и команд
dropdb имяБД - удаляет БД
\du - покажет таблицу с именами и правами пользователей
ALTER USER postgres WITH PASSWORD '1q2w3e4r'; - меняет пароль суперюзера
ALTER ROLE - будет означать, что пароль изменен
CREATE USER new_user WITH PASSWORD '1q2w3e4r'; - создает нового пользователя и пароль для него
CREATE ROLE - будет означать, что новый пользователь создан
ALTER USER new_user WITH SUPERUSER; - дает права суперюзера пользователю new_user
ALTER ROLE - будет означать, что успешно изменены права пользователя
postgres=# DROP USER new_user; - удаляет пользователя new_user
DROP ROLE - будет означать, что пользователь успешно удален
postgres@alex-Veriton-S2610G:~$ exit - выход
postgres@alex-Veriton-S2610G:~$ man psql - покажет все команды
